Implement Soft Delete

Published on Sunday, July 15th 2018 at 20:34
Last updated on Monday, December 17th 2018 at 20:56

Issue

This morning, our boss requires all the deletions of data to be soft delete (only mark them as deleted). I searched a lot about how to do it. Here is my way to implement it.

Analysis

Soft deletion means two things: 1. Each entity has a boolean flag “isDeleted”. When delete, we mark it as deleted 2. when retrieved data, ignore those ‘deleted’ items The most easy solution I had was to rewrite delete() method of JPA repositories. Then, we need to do condition check in each query. That’s dumb.

Solution

Gratefully, JPA has annotations we can use: @SQLDelete and @Where @SQLDelete tells JPA what kind of sql sentence you want to execute when delete() function is called. We can redefine the delete action here. @Where is the condition you want to append to the query of this entity The code will be like:

@Data
@Entity
@Table(name = "tb_tasks")
@SQLDelete(sql = "UPDATE tb_tasks SET is_deleted = true WHERE task_id = ?")   // Soft delete
@Where(clause = "is_deleted IS NULL OR is_deleted = false")
public class Task extends CuraPatientEntity {
....
...
@Column ("is_deleted")
private Boolean isDeleted;

}

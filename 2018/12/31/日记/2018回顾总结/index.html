2018 回顾总结

Posted on 2018-12-31 | Edited on 2019-01-15 | In Daily

现在是洛杉矶时间2018年12月31日10:32， 北京时间2019年1月1日2:32
美国时间一年的末尾，中国时间新一年的开始，在这个特殊的时间段，回顾一下自己的2018,。

2018

回顾

总的来说2018年是非常宅的一年，我从来没这么宅过, 也从来没有这么低沉过。

2018的年初一直在家待着，完成网课等待毕业并且在找工作的焦虑中度过。 那个时候，感觉到自己要留下一些东西，留下一些学习和生活过的痕迹。于是开始了自己的博客，一开始是用WordPress搭的，后期转成了Hexo的静态博客。

4月份的时候，遇到了人生中的第一份软件开发工作，并且孤身一人从SD搬到了Irvine，生活变得更加孤独。在工作中，一开始接受的项目代码质量太差了，差点给我劝退了，到后来接受了新的项目，才一点点好起来。但自己也明显感觉到，能在这家公司学到的东西实在太有限了，到现在为止，工作中已经没有什么难点了，全部是需求在来来回回地变化，自己也进入了一个舒适区，对自己发展来说非常危险。特别希望一个有经验的Tech Leader来指导我，那我的成长一定是十分迅速的。因此年底的时候，开始投简历想要跳槽。但整个找工作过程并没有那么认真，可能是因为进入了舒适区，没有那么努力（其实一直觉得自己不够努力，只是表现得自己很努力）。后来得到一个非常好的内推机会，走到最后Onsite的时候才知道，自己离得这么远，虽然take home project过了， 但是当场写算法和思考的能力太差了， 欸，还是怪自己没有那么努力地去做题和钻研。

2018年其实是从学生生涯进入职业生涯的转变期，学生生涯都靠着写写作业和临时抱佛脚，都还能不错的应付过去了。 真正离开学校后发现，再也不能像学生一样等课程给自己安排任务，需要自己摸索发展方向，更加主动地去学习。 2018年一直在与自己的惰性做斗争，想要早睡早起，想要多读书，想要多学东西，但是都进行得不怎么样。

然后情感生活就不在这里说了，心中有数，要先让自己强大起来。

收获

做人呐，总不能那么负面，虽然2018过得没那么顺利，没那么开心，但总是有收获的。

  • 对数据库的使用没有那么陌生了
    因为没有上过数据库的课程，一直对数据库这一块非常陌生和抵触。但由于工作关系，需要管理整个项目的源数据sql，平时要写很多Query，所以对MySQL和PostgreSQL更加熟悉了。
  • 学习了框架：Spring JPA， Angular， SpringBoot
  • 开发工具的熟悉：Git， Flyway， Docker等
  • 算法的进步不大，但是也有一些
  • 并发编程
    平时关注的比较多，看的书也是关于并发编程的

看的书籍

  • Effective Java
  • Concurrent Programming in Java
  • 大话设计模式
  • 东野圭吾 - 秘密
  • 东野圭吾 - 嫌疑人X的献身
  • 东野圭吾 - 恶意
  • 东野圭吾 - 梦幻花
  • 东野圭吾 - 毕业-雪月花杀人游戏
    看了太多小说，到年底才意识到要多看技术书籍, 2019多读一些技术书籍，同时读一些拓展视野的书籍。

2019

对于2019年何去何从暂时还打个问号，可能回国也可能还在美国待一段时间，但是在美国我一点也不开心。
2019年，希望自己能做到以下几点，应该就能收获很多了：

  • 多读书
  • 切勿浅尝辄止
  • 多问自己为什么，而不是搜索引擎
    不要脱发！

学习清单

  • 读完 Java Concurrency In Practice
  • 学习一个Non-relational Database
  • 阅读Java源码 （部分）：学习代码风格，了解接口设计
  • LeetCode第二遍刷完 （并非所有题目）： 为了找工作，哈哈
  • 学习 JVM: 了解内存机制还有一些多线程的实现方法
  • 学习 Scala：熟悉函数式编程
  • 完成Java的爬虫框架项目： 多线程和设计模式的练习，更加熟悉Http

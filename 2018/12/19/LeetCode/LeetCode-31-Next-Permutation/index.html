LeetCode 31. Next Permutation

Posted on 2018-12-19 | Edited on 2019-01-15

这是LeetCode系列的第一篇博客，也在尝试改变自己做题的方法，知道解法不等于理解。特别是没做出来的题目，看了解法之后感叹“好神奇啊”，这种时候得静下心来细细分析了，并且尝试正推，想一想正着是如何推导出来的。

题目

31.Next Permutation
Implement next permutation, which rearranges numbers into the lexicographically next greater permutation of numbers.

If such arrangement is not possible, it must rearrange it as the lowest possible order (ie, sorted in ascending order).

The replacement must be in-place and use only constant extra memory.

Here are some examples. Inputs are in the left-hand column and its corresponding outputs are in the right-hand column.

1,2,3 → 1,3,2
3,2,1 → 1,2,3
1,1,5 → 1,5,1

思路

准备

什么是lexicographically: 字典顺序
开始：从左到右递增
结束：从左到右递减
个人的Notation： ()表示集合， []表示一种带顺序的排列

分析

先找一下字典顺序的规律，比如说数组: [1, 2, 3, 4, 5]，开始值是从左到右递增，最大是从左到右递减。 方便分析，直接用十进制的值来表达，还有一点就是，为了让值一点点递增，我们是要尽量先修改右边权值比较小的位数。我用变动的位数来分组，在相同变动的位数中，不同的排序会产生不同的值，可以暂时忽略前面共同的部分，尝试寻找规律：

变动1位

12345， 因为每次变动是数字位置调换，只变动一位是不可能的

变动2位 (4, 5)

45
54
把大一点的数字换到值高一点的位置上，暂时还看不出规律。 两位不够用了，需要借3的位置

变动3位 (3, 4, 5)

354
这里可以进入一个子问题: 集合(3, 4, 5)从[3, 5, 4]开始的字典顺序
因为已知[5, 4] 已经是只变动两位的最大值了,需要挑选一个值放在3的位置上。为了避免向前跨的步数过大，需要挑选最小的比3大的值。所以把4放在最前面，然后要确保后两位是从最小值开始，如前面所说，最小值就是递增,可以得到:[4, 3, 5]
因为后两位[3, 5]也是有上升的空间，然后我们又可以进入到一个子问题：集合(3, 5)从[3, 5]开始的字典顺序
我们知道[3, 5]这两位下一个值是[5,3]， 然后就达到了这两位的最大值, 我们返回到三位也就是：[4, 5, 3]。 后面位数值达到最大的时候，我们尝试动最前面一位，这样不断重复下去，直到3位不够用了.
全部列出来就是：
354
435
453
534
543
到这里，(3, 4, 5) 这个集合达到了最大值，也就是[5, 4, 3], 从左到右递减，接着尝试借用2的位置.

变动4位 (2, 3, 4, 5)

开始：2543 （除了2， 其余部分递减）
2534
2543
….
5432 (递减)

变动5位 (1, 2, 3, 4, 5)

15432 （除了1， 其余部分递减）
….
到最后为达到全部从左到右递减：[5, 4, 3, 2, 1]

归纳

其实这样分析是在尝试理解什么是字典顺序，到这里其实对本题的解法还没有太大帮助，但是可以理解其中的原理，又是分组又是子问题的，搞得很复杂, 尝试归纳一下上面过程中重要的逻辑点（粗体部分）:

  1. 当最右边k位数是递减时，最右边k位数的排列已经达到最大值了，需要向左边借位，可以写成: 当[i, end] 是递减时，需要用i-1的位置
  2. 选择放在i-1位置上值的规律：选择[i, end] 当中，比nums[i-1]大的最小数字
  3. 最高位i-1上已经换上一个稍大一点的值了，然后把[i, end]的排列最小化，也就是从左到右递增
    最重要的理解来了，也是困扰我很久的部分，之前不知道这段分析有什么用，不知道如何运用这个规律，其实当我们把[i, end]从左到右递增排序后（上面第三步），整个问题就变成了最右一位数是递减(只有一位，可以理解成递减), 最右一位达到最大值，返回第一步重新套用这个寻找规则

解题

  1. 从右到左迭代，直到nums[i] > nums[i-1]。可以知道[i, end] 是递减
  2. 从[i, end]中找到比nums[i-1]大的最小值nums[j], 对调nums[i]和nums[j]
  3. 翻转[i, end]， 让它升序
